# Muscle App – 達成度をゲーミフィケーションで高める体型管理アプリ

## 作ろうと思ったきっかけ
大学のラクロス部でトレーニングリーダーをしており、  
「みんなの筋トレのモチベーションが上がる仕組みを作りたい」  
という思いがスタートでした。

既存の筋トレアプリは「記録ができる」「タイマーがある」などの  
最低限の機能が中心で、楽しみながら継続する工夫が少なく感じていました。

そこで、

- 目標の体型を選ぶ  
- 現在の体データから“達成レベル”を数値化  
- 成長をグラフで可視化  
- 称号（ランク）でゲーム感覚のフィードバック  

といった仕組みを自分で考え、  
「仲間や自分自身のモチベーションが上がるアプリ」を目指して制作しました。

---

## 主な機能

- **ユーザー登録・ログイン（JWT 認証）**
- **目標体型プリセット選択（4種類）**
  - 悟空タイプ / スマートアスリートタイプ / パワーリフタータイプ / カスタム
- **身長・体重・体脂肪率から BMI と達成度（0〜100）を計算**
- **計算結果をユーザーごとにデータベースへ保存**
- **レベル推移を折れ線グラフで可視化（Chart.js）**
- **未ログインユーザーは「計算のみ可能」・履歴は非表示**

---

## 使用技術

### Backend
- Python 3
- FastAPI
- SQLAlchemy + SQLite (muscle_app.db)
- JWT 認証（OAuth2PasswordBearer）

### Frontend
- HTML / CSS / JavaScript
- Chart.js（折れ線グラフ）

## ディレクトリ構成（抜粋）

```bash
muscle-app/
├ backend/
│  ├ main.py        # エントリポイント（API 定義）
│  ├ auth.py        # 認証・ログイン処理
│  ├ db.py          # DB 接続・初期化
│  ├ models.py      # User, Measurement モデル
│  ├ presets.py     # プリセット体型の定義
│  └ schemas.py     # Pydantic スキーマ
└ frontend/
   ├ index.html     # 目標体型選択画面
   ├ login.html     # ログイン画面
   ├ input.html     # 体データ入力・レベル計算・グラフ表示画面
   ├ style.css      # スタイルシート
   └ script.js      # トップ画面の JS

起動方法
# （初回のみ）依存ライブラリのインストール
pip install -r requirements.txt

# backend ディレクトリでサーバー起動
cd backend
uvicorn main:app --reload


ブラウザで
http://127.0.0.1:8000/

にアクセスするとトップ画面が表示されます。


##使い方

http://127.0.0.1:8000/docs
 を開き、POST /auth/register でユーザー登録

frontend/login.html からログイン

トップ画面で目標体型を選択

入力画面で身長・体重・体脂肪率を入力し「レベルを計算」

ログイン済みの場合、結果が DB に保存され、
前回の記録・レベル推移グラフとして表示

別アカウントでログインすると、そのユーザー専用の履歴が表示される

##今後の拡張アイデア / ロードマップ

・本プロジェクトは、以下の機能拡張を想定し、継続開発できる設計としています。

・プリセット体型の追加・編集機能
理想体型をユーザー自身が作成・共有できる仕組み。

・トレーニングメニューの自動提案（LLM 連携）
現在のレベル・目標・可処分時間をもとに、
LLM が最適なトレーニングメニューを生成する。

・1 日の記録を複数保存し、日・週・月単位で集計するダッシュボード

・スマホ最適化デザイン（モバイルファースト UI）
実際の利用シーン（ジム・部活）を想定した操作性改善。

・InBody 等の体組成レポートの読み込み（PDF/画像解析）

　　・身体項目を自動抽出

　　・理想値と比較し不足分を可視化

　　・「どこを鍛えるべきか」「週何回何をすべきか」を具体的に提案

アプリを “記録するだけのツール” から “データに基づくコーチ” へ発展させることを目指しています。
